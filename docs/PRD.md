# 表情派（EmojiPie）产品需求文档 PRD

## 1. 产品概述

## 1.1 产品名称

中文名称：

表情派

英文名称：

EmojiPie

---

## 1.2 产品定位

表情派是一款 AI 表情生成工具。

用户输入一句话，系统理解用户表达的情绪、语气和场景，自动生成符合内容的聊天表情图片。

产品核心能力：

```

一句话
↓
理解情绪
↓
生成专属表情
↓
复制发送

```

---

## 1.3 产品目标

帮助用户快速生成个性化聊天表情，降低寻找表情和制作表情的成本。

目标场景：

- 日常聊天回复
- 工作沟通
- 朋友互动
- 情绪表达
- 网络玩梗

---

## 4.3.5 输出形式与图片设置

提供三种互斥的输出形式：

- 行内 Emoji：输出 5 个标准 Unicode 文本候选，不生成图片。
- 黄脸贴纸：输出 9 张 `128x128` 透明 PNG，主体突出。
- 表情海报：输出 9 张 `640x640` PNG，保留背景与现有海报构图。

新用户默认选择黄脸贴纸。选择行内 Emoji 时隐藏图片内文字和图片效果；切回图片输出时恢复
最近一次图片版式与文字设置。关闭图片内文字后，caption 仍用于卡片标题、搜索与导出文件名。
切换输出形式时清除当前候选，已生成图片仍保留在历史中。再次创作图片历史记录时恢复该
记录的版式与文字设置。

---

# 4.4 结果展示

## 展示方式

每次只展示当前输出形式的结果：行内 Emoji 固定 5 个候选，黄脸贴纸或表情海报固定 9 张。
行内 Emoji 点击后直接写入文本剪贴板；微信、飞书、钉钉等客户端负责按各自系统字体渲染。
生成完成后自动将视野定位到结果区。

页面根据窗口大小调整候选网格列数，但不自动生成或追加结果。

示例：

```

选择一种输出形式

↓

生成 5 个行内 Emoji 或 9 张图片

↓

复制当前类型的结果

```

---

## 候选刷新

用户点击“换一批”时：

```

保留当前类型的候选

↓

按当前批次快照重新生成整组候选

↓

成功后整体替换

```

刷新失败时保留原候选。图片批次继续保存在最近生成中；行内 Emoji 只保留当前批次，
不进入图片历史与收藏库。

---

## 结果去重

避免：

- 相同表情重复
- 相同文字重复
- 相同组合重复

# 4.5 表情操作

支持：

点击黄脸贴纸或表情海报：

```

复制图片到剪贴板

## 换一批

用户点击：

```

```

系统保持：

- 原始输入
- 当前模式
- 当前输出形式
- 图片模式下的当前效果
- 图片模式下的版式与图片内文字设置

重新执行生成流程并替换当前整组候选。

# 4.6 风格选择

MVP 支持真实缩略图选择与以下效果：

- 智能搭配
- 经典黄脸

- 冷淡敷衍
- 社畜打工
- 阴阳怪气
- 吃瓜围观

智能搭配根据情绪、场景和候选位置，为一批结果选择多种具体效果。生成记录只保存实际
使用的效果，确保历史记录可以稳定复现。

后续扩展：

# 5. AI 生成能力

## 5.1 文本理解

分析输入内容：

- 情绪
- 场景
- 关键词
- 语气
- 表达目的

示例：

输入：

```

这个需求很简单

````

分析：

```json
{
  "emotion": "无奈",
  "scene": "工作",
  "tone": "吐槽",
  "intent": "表达压力"
}
````

---

# 5.2 表情生成参数

生成结构：

```json
{
  "character": "yellow_face",
  "emotion": "崩溃",
  "action": "抱头",
  "object": "电脑",
  "caption": "简单改一下",
  "style": "office"
}
```

---

# 5.3 回复模式生成

输入：

```
今晚加班吗？
```

生成：

多个回复方向：

```
好的，我已经准备好了

今晚又是属于代码的一天

身体可以拒绝，需求不允许
```

再转换为表情。
