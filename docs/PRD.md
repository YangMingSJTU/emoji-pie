# 表情派（EmojiPie）产品需求文档 PRD

## 1. 产品概述

## 1.1 产品名称

中文名称：

表情派

英文名称：

EmojiPie

---

## 1.2 产品定位

表情派是一款 AI 表情生成工具。

用户输入一句话，系统理解用户表达的情绪、语气和场景，自动生成符合内容的聊天表情图片。

产品核心能力：

```

一句话
↓
理解情绪
↓
生成专属表情
↓
复制发送

```

---

## 1.3 产品目标

帮助用户快速生成个性化聊天表情，降低寻找表情和制作表情的成本。

目标场景：

- 日常聊天回复
- 工作沟通
- 朋友互动
- 情绪表达
- 网络玩梗

---

# 2. 产品核心理念

## 2.1 核心表达

将文字转换为视觉化表达。

示例：

输入：

```

今天又要加班

```

生成：

```

黄脸角色

表情：
疲惫
黑眼圈
生无可恋

动作：
趴在电脑前

文字：
今天又要加班

```

---

## 2.2 产品特点

- AI 自动理解文字含义
- 自动匹配情绪
- 自动生成表情元素
- 图片内文字可选
- 快速复制使用

# 4.2 生成模式

## 模式一：表达模式

---

## 4.3.5 版式与图片内文字

提供两种版式：

- 小黄脸：默认输出 `256x256` 透明 PNG，主体突出，适合聊天窗口中的小尺寸发送。
- 表情海报：输出 `640x640` PNG，保留背景与现有海报构图。

用户可以独立控制是否将 caption 绘制到图片内。默认关闭；关闭后 caption 仍用于卡片标题、
搜索与导出文件名。应用记住最近一次版式与文字选择，再次创作历史记录时恢复对应设置。

# 5. AI 生成能力

## 5.1 文本理解

分析输入内容：

- 情绪
- 场景
- 关键词
- 语气
- 表达目的

示例：

输入：

```

这个需求很简单

````

分析：

```json
{
  "emotion": "无奈",
  "scene": "工作",
  "tone": "吐槽",
  "intent": "表达压力"
}
````

---

# 5.2 表情生成参数

生成结构：

```json
{
  "character": "yellow_face",
  "emotion": "崩溃",
  "action": "抱头",
  "object": "电脑",
  "caption": "简单改一下",
  "style": "office"
}
```

---

# 5.3 回复模式生成

输入：

```
今晚加班吗？
```

生成：

多个回复方向：

```
好的，我已经准备好了

今晚又是属于代码的一天

身体可以拒绝，需求不允许
```

再转换为表情。
